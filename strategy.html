Patrón de Diseño Strategy

¿Qué es el Strategy?

El patrón Strategy es un patrón de comportamiento que permite definir una familia de algoritmos, encapsular cada uno y hacerlos intercambiables. Strategy permite cambiar el comportamiento de un objeto en tiempo de ejecución.

¿Qué problema resuelve?

Evita el uso excesivo de condicionales para seleccionar comportamientos. Cada estrategia representa una forma diferente de ejecutar una operación, permitiendo mayor flexibilidad y mantenimiento del código.

Casos de uso

  • Implementar algoritmos que puedan variar (ej. compresión, ordenamiento).
  • Seleccionar dinámicamente un algoritmo en tiempo de ejecución.
  • Evitar estructuras de control complejas en código.

Ejemplo de código en Java

interface Estrategia {
  void ejecutar();
}

class EstrategiaA implements Estrategia {
  public void ejecutar() {
    System.out.println("Estrategia A ejecutada");
  }
}

class EstrategiaB implements Estrategia {
  public void ejecutar() {
    System.out.println("Estrategia B ejecutada");
  }
}

class Contexto {
  private Estrategia estrategia;

  public void setEstrategia(Estrategia estrategia) {
    this.estrategia = estrategia;
  }

  public void ejecutarEstrategia() {
    estrategia.ejecutar();
  }
}

Ventajas

  • Permite cambiar algoritmos en tiempo de ejecución.
  • Reduce el acoplamiento entre contexto y algoritmo.
  • El código es más limpio y fácil de mantener.

Desventajas

  • Aumenta el número de clases.
  • Puede introducir complejidad innecesaria para comportamientos simples.
