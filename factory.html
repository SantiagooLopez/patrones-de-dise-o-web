Patrón Factory Method

El patrón Factory Method es un patrón de diseño creacional que proporciona una interfaz para crear objetos en una superclase, pero permite a las subclases alterar el tipo de objetos que se crean.

¿Qué problema resuelve?

Evita el acoplamiento directo del código cliente con clases concretas, promoviendo el uso de interfaces y permitiendo la extensión sin modificar el código existente.

Casos de uso

  • Cuando el código debe trabajar con objetos que todavía no conoce su clase exacta
  • Al delegar la responsabilidad de creación de objetos a subclases
  • En sistemas que necesitan alta extensibilidad

Ventajas

  • Desacopla la lógica de creación de la lógica de uso
  • Fácil de añadir nuevos tipos sin tocar código existente
  • Promueve el uso de interfaces y clases abstractas

Ejemplo en Java

// Producto
interface Producto {
  void operacion();
}

// Producto concreto
class ProductoConcreto implements Producto {
  public void operacion() {
    System.out.println("Operación del producto concreto");
  }
}

// Creador
abstract class Creador {
  public abstract Producto crearProducto();
}

// Creador concreto
class CreadorConcreto extends Creador {
  public Producto crearProducto() {
    return new ProductoConcreto();
  }
}
