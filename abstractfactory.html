Patrón Abstract Factory

El patrón Abstract Factory es un patrón creacional que proporciona una interfaz para crear familias de objetos relacionados o dependientes sin especificar sus clases concretas.

¿Qué problema resuelve?

Este patrón evita la dependencia directa del código cliente con las clases concretas, permitiendo el uso de interfaces para instanciar familias de objetos que trabajen bien juntas.

Casos de uso

  • Diseños donde se usan múltiples familias de productos relacionados
  • Herramientas gráficas multiplataforma (crear botones y ventanas distintos según sistema operativo)
  • Sistemas que deben ser independientes del sistema de objetos que utilizan

Ventajas

  • Asegura la compatibilidad entre objetos
  • Facilita el cambio de familias de productos fácilmente
  • Cumple con el Principio de Inversión de Dependencias

Ejemplo en Java

// Interfaces
interface Boton { void dibujar(); }
interface Ventana { void abrir(); }

// Productos concretos
class BotonMac implements Boton {
  public void dibujar() { System.out.println("Botón estilo Mac"); }
}
class VentanaMac implements Ventana {
  public void abrir() { System.out.println("Ventana estilo Mac"); }
}

// Abstract Factory
interface GUIFactory {
  Boton crearBoton();
  Ventana crearVentana();
}

// Fábrica concreta
class MacFactory implements GUIFactory {
  public Boton crearBoton() { return new BotonMac(); }
  public Ventana crearVentana() { return new VentanaMac(); }
}
