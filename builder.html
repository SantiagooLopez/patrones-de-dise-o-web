Patrón de Diseño Builder

¿Qué es el Builder?

El patrón Builder es un patrón de diseño creacional que permite construir objetos complejos paso a paso. Separa la construcción de un objeto de su representación, de modo que el mismo proceso de construcción puede crear diferentes representaciones.

¿Qué problema resuelve?

Builder es útil cuando una clase tiene muchos parámetros opcionales o cuando el proceso de construcción es complejo. En lugar de usar múltiples constructores, se define un objeto Builder que configura el objeto paso a paso.

Casos de uso

  • Creación de objetos con muchos parámetros opcionales.
  • Construcción de objetos con múltiples etapas o configuraciones.
  • Construcción de interfaces de usuario o estructuras anidadas.

Ejemplo de código en Java

public class Auto {
  private String marca;
  private String modelo;
  private int año;

  private Auto(Builder builder) {
    this.marca = builder.marca;
    this.modelo = builder.modelo;
    this.año = builder.año;
  }

  public static class Builder {
    private String marca;
    private String modelo;
    private int año;

    public Builder setMarca(String marca) {
      this.marca = marca;
      return this;
    }

    public Builder setModelo(String modelo) {
      this.modelo = modelo;
      return this;
    }

    public Builder setAño(int año) {
      this.año = año;
      return this;
    }

    public Auto build() {
      return new Auto(this);
    }
  }
}

Ventajas

  • Facilita la creación de objetos complejos paso a paso.
  • Mejora la legibilidad del código.
  • Evita constructores con muchos parámetros.

Desventajas

  • Puede requerir más código en comparación con un constructor simple.
  • Puede ser excesivo para objetos simples.
