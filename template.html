Patrón Template Method

El patrón Template Method define el esqueleto de un algoritmo en una operación, delegando algunos pasos a las subclases. Permite redefinir ciertos pasos de un algoritmo sin cambiar su estructura general.

¿Qué problema resuelve?

Facilita la reutilización del código común a través de clases abstractas, y permite personalizar partes específicas del algoritmo en subclases concretas.

Casos de uso

  • Procesamiento de archivos con formato común
  • Flujos de trabajo predefinidos con pasos personalizables
  • Algoritmos que requieren pasos invariantes y variantes

Ventajas

  • Evita la duplicación de código
  • Facilita la extensión del algoritmo
  • Aplica el principio de inversión de control

Ejemplo en Java

// Clase abstracta
abstract class Juego {
  void jugar() {
    iniciar();
    jugarTurno();
    terminar();
  }
  abstract void iniciar();
  abstract void jugarTurno();
  abstract void terminar();
}

// Subclase concreta
class Ajedrez extends Juego {
  void iniciar() { System.out.println("Inicia Ajedrez"); }
  void jugarTurno() { System.out.println("Moviendo fichas..."); }
  void terminar() { System.out.println("Fin del juego"); }
}
