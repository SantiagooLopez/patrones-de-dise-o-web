Patrón Prototype

El patrón Prototype es un patrón de diseño creacional que permite copiar objetos existentes sin acoplar el código a sus clases concretas. Se basa en clonar un objeto prototipo para crear nuevos objetos.

¿Qué problema resuelve?

Permite crear nuevos objetos de forma eficiente a partir de una instancia prototipo, especialmente cuando la creación es costosa o complicada.

Casos de uso

  • Cuando los objetos son costosos de crear
  • Para evitar la repetición de configuración compleja
  • Al generar múltiples objetos con las mismas propiedades

Ventajas

  • Reduce la necesidad de subclases adicionales
  • Permite crear copias exactas de objetos complejos
  • Menor acoplamiento que con el uso de constructores directos

Ejemplo en Java

interface Prototipo extends Cloneable {
  Prototipo clonar();
}

class Documento implements Prototipo {
  private String contenido;

  public Documento(String contenido) {
    this.contenido = contenido;
  }

  public Prototipo clonar() {
    return new Documento(this.contenido);
  }
}
