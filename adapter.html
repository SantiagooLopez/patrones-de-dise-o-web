Patrón de Diseño Adapter

¿Qué es el Adapter?

El patrón Adapter es un patrón estructural que permite que dos interfaces incompatibles trabajen juntas. El adaptador actúa como un puente entre ambas, traduciendo llamadas de una a otra.

¿Qué problema resuelve?

Cuando se necesita integrar una clase con una interfaz diferente a la esperada por el cliente, el patrón Adapter facilita la compatibilidad sin modificar el código existente.

Casos de uso

  • Integrar sistemas antiguos con nuevos.
  • Adaptar bibliotecas de terceros a tu código.
  • Unificar interfaces diversas bajo un mismo modelo.

Ejemplo de código en Java

// Clase incompatible con nuestra interfaz
class EnchufeEuropeo {
  public void conectar() {
    System.out.println("Conectado a enchufe europeo.");
  }
}

// Interfaz esperada
interface EnchufeAmericano {
  void enchufar();
}

// Adaptador
class Adaptador implements EnchufeAmericano {
  private EnchufeEuropeo europeo;

  public Adaptador(EnchufeEuropeo europeo) {
    this.europeo = europeo;
  }

  public void enchufar() {
    europeo.conectar();
  }
}

Ventajas

  • Permite reutilizar clases existentes sin modificar su código.
  • Facilita la integración entre sistemas con interfaces distintas.

Desventajas

  • Puede agregar complejidad adicional.
  • El uso excesivo puede generar dificultades para mantener el código.
