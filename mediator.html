Patrón Mediator

El patrón Mediator es un patrón de comportamiento que define un objeto que encapsula cómo interactúan un conjunto de objetos, promoviendo un bajo acoplamiento entre ellos.

¿Qué problema resuelve?

Evita que los objetos se refieran entre sí explícitamente, facilitando la reutilización individual y la independencia de componentes.

Casos de uso

  • Interfaces gráficas con múltiples componentes interactivos
  • Chats, donde un servidor centraliza la comunicación
  • Controladores entre módulos independientes

Ventajas

  • Reduce el acoplamiento entre componentes
  • Centraliza la lógica de interacción
  • Facilita el mantenimiento y la escalabilidad

Ejemplo en Java

// Mediador
interface ChatMediator {
  void enviarMensaje(String mensaje, Usuario usuario);
}

// Usuario
class Usuario {
  private String nombre;
  private ChatMediator mediador;

  public Usuario(String nombre, ChatMediator mediador) {
    this.nombre = nombre;
    this.mediador = mediador;
  }

  public void enviar(String mensaje) {
    mediador.enviarMensaje(mensaje, this);
  }

  public String getNombre() {
    return nombre;
  }
}
