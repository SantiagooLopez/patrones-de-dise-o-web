Patrón de Diseño Observer

¿Qué es el Observer?

El patrón Observer es un patrón de comportamiento que permite definir una dependencia uno-a-muchos entre objetos. Cuando un objeto cambia su estado, todos sus dependientes son notificados automáticamente.

¿Qué problema resuelve?

Es útil cuando el cambio de estado de un objeto debe reflejarse en otros objetos sin acoplar estrechamente sus clases. Permite una arquitectura más flexible y dinámica.

Casos de uso

  • Interfaces gráficas (actualizar vistas cuando cambia el modelo).
  • Sistemas de suscripción y notificación.
  • Monitoreo de eventos en tiempo real.

Ejemplo de código en Java

import java.util.*;

interface Observador {
  void actualizar(String mensaje);
}

class Sujeto {
  private List observadores = new ArrayList<>();

  public void agregar(Observador o) {
    observadores.add(o);
  }

  public void notificar(String mensaje) {
    for (Observador o : observadores) {
      o.actualizar(mensaje);
    }
  }
}

class Usuario implements Observador {
  private String nombre;
  public Usuario(String nombre) {
    this.nombre = nombre;
  }
  public void actualizar(String mensaje) {
    System.out.println(nombre + " recibió: " + mensaje);
  }
}

Ventajas

  • Reduce el acoplamiento entre los objetos.
  • Permite la suscripción y notificación dinámica.
  • Facilita la reutilización y escalabilidad.

Desventajas

  • Puede generar complejidad si hay muchos observadores.
  • Difícil de depurar cuando hay notificaciones múltiples.
